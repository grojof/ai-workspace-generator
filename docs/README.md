# Documentación

Estructura de la documentación de este repositorio. Los **nombres de carpeta son estables (en inglés)**
para ser predecibles para herramientas y para la IA; el **contenido** está en español.

- **[`project/`](project/README.md)** — documentación del proyecto (para personas): la **[guía de uso](project/USAGE.md)**
  (CLI + config + multi-repo), arquitectura, cómo extender y mantener, distribución y las decisiones (ADR).
- **[`development/`](development/README.md)** — proceso de desarrollo (mantenido con IA):
  - `specs/` — especificaciones vigentes (la verdad actual del comportamiento).
  - `changes/` — cambios SDD en curso (`changes/archive/` para los completados).
  - `status/` — foto viva del proyecto (`PROJECT-STATE.md`); se refresca con `/doc-sync`.

> Empieza por el **[README raíz](../README.md)** (resumen) o por **[project/](project/README.md)** para el detalle.
